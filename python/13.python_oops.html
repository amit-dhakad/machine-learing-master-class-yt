Understanding how Python constructs objects, manages state with 'self', and handles the lifecycle of an instance.

1.1

Class vs Object (The Definition)

A Class is a user-defined prototype for an object that defines a set of attributes that characterize any object of the class. An Object is a unique instance of a data structure that's defined by its class.

CLASS
(The Blueprint)
OBJECT A
(Instance 1)
OBJECT B
(Instance 2)
📱 Real World Analogy: Smartphone
Think of a Class as the factory specifications for an iPhone 15. It says it must have a screen, a battery, and a camera.

When you buy the phone, the physical device in your hand is the Object. Your phone might have 10% battery (State), while mine has 90%. They are both the same "Class" but different "Objects".
basic_class.py 
class Smartphone:
    # The Constructor
    def __init__(self, model, battery):
        self.model = model      # Attribute
        self.battery = battery  # Attribute

    def check_status(self):
        print(f"{self.model} is at {self.battery}%")

phone1 = Smartphone("iPhone 15", 85)
phone1.check_status()
1.2

Deep Dive: The 'self' Pointer

In Python, self is not just a variable name; it is the Identity of the object. It acts as a bridge between the class blueprint and the specific memory instance.

1. The Explicit-Passing Mechanism

In other languages like Java or C++, the pointer (often called this) is hidden. In Python, it is explicit. When you call my_phone.call(), Python internally transforms it into:
Smartphone.call(my_phone).
This is why every instance method must accept self as the first argument.

2. Inter-Method Communication

self allows one method to call another method within the same object. This is essential for breaking down complex logic into smaller, reusable parts.

instance_state.py 
class IdentityCheck:
    def get_id(self):
        print(f"Memory ID of self: {id(self)}")

    def method_a(self):
        print("Inside A")
        self.method_b()  # Calling B from A using self

    def method_b(self):
        print("Inside B")

obj = IdentityCheck()
print(f"Memory ID of obj:  {id(obj)}")
obj.get_id()  # Both IDs will match exactly!
📍 The Address Analogy
If you are inside a house and someone asks "What color is the wall?", you know they mean this house. self is like the address of the house. It tells Python exactly which object's memory location to look at. Without self, Python would be like a person in a neighborhood not knowing which house to enter.
1.3

Deep Theory: Instance, Class & Static Methods

In Python, not all methods act the same way. The Decorator you use changes how the method is "bound" to the class or the object. Understanding this is key to building scalable API designs.

Method Type First Argument Binding Behavior Primary Purpose
Instance self Bound to the Instance Modifying specific object state.
Class cls Bound to the Class Factory methods & Class-wide state.
Static None Unbound (Like a function) Utilities & Namespace organization.

1. Instance Methods: The Object's Controller

These are the "default" methods. When you call obj.method(), Python automatically passes the object itself as the first argument (self). This allows the method to read and write attributes unique to that specific instance.
Binding: They are bound methods, meaning they "carry" the instance they belong to.

2. Class Methods: The Factory Pattern

Defined with @classmethod, these receive the class itself (cls) as the first argument. They are widely used for Alternative Constructors.
Scenario: You have a User class. Sometimes you want to create a user from a db_row, sometimes from a json_string, and sometimes from a csv_line. Instead of making __init__ super complex, you create from_json() and from_csv() class methods.

3. Static Methods: Clean Namespacing

Defined with @staticmethod, these don't receive self or cls. They are essentially just functions that happen to live inside a class.
Scenario: If you have a Math class, you might want a is_even() helper. It doesn't need to know anything about the class state; it's just a utility that belongs in the Math folder (Namespace).

professional_method_patterns.py 
class Book:
    library_name = "Central Library"

    def __init__(self, title, author):
        self.title = title
        self.author = author

    # Instance Method
    def get_details(self):
        return f"'{self.title}' by {self.author}"

    # Class Method: Factory (Alternative Constructor)
    @classmethod
    def from_isbn(cls, isbn):
        # Logic to fetch title/author from an API
        title, author = "Automate boring stuff", "Al Sweigart"
        return cls(title, author)

    # Static Method: Generic Utility
    @staticmethod
    def is_valid_isbn(isbn):
        return len(isbn) == 13

# Usage
b1 = Book.from_isbn("9781234567890")  # Factory in action
print(Book.is_valid_isbn("123"))      # Static Utility in action
🚀 Senior Tip: When to use which?
Ask yourself: "Does this method need to know about the instance (self)?" If yes, use Instance Method.
If no, ask: "Does it need to know about the class (cls)?" If yes, use Class Method.
If it needs neither, use a Static Method or just a Module-level function.

The Constructor Deep Dive: __init__

The constructor is a special method that Python automatically calls when an object is created. Its primary job is to assign values to the data members (attributes) of the class.

Class() Called
→
__new__
(Memory Alloc)
→
__init__
(State Setup)
→
Object Ready

Types of Constructors

Type Description
Default Automatically provided by Python if you don't define __init__. It does nothing.
Non-Parameterized Explicitly defined but only takes self. Used to set fixed default values.
Parameterized Takes custom arguments (like name, age) to initialize the object uniquely.

The "Constructor Overloading" Myth

Unlike Java or C++, Python does not support multiple __init__ methods. If you define two, the second one will overwrite the first.
The Solution: Use default arguments or *args / **kwargs to make a single constructor flexible.

constructor_lifecycle.py 
class Profile:
    # 1. Parameterized with Default Values (Flexible)
    def __init__(self, name, role="Guest", **extra_info):
        self.name = name
        self.role = role
        self.meta = extra_info  # Captures any additional data

    def show(self):
        print(f"{self.name} ({self.role}) - Data: {self.meta}")

# Case A: Minimal setup
p1 = Profile("Amit") 
# Case B: Full setup with unlimited dynamic data
p2 = Profile("Sunda", "Admin", city="Mumbai", xp=10)
📝 Real World Analogy: Joining a Gym
When you walk into a gym for the first time, you are asked to fill out a form (Name, Age, Plan). This is the Constructor. You can't start using the machines until your profile is "initialized" with your data.

If you don't fill out the form, you don't have an "instance" of a membership.
