Module 02 · Polymorphism

The Power of Polymorphism

"Poly" (Many) + "Morph" (Forms). One interface, multiple implementations. This is the heart of flexible software architecture.

01 What is Polymorphism?

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It means the same function name can be used for different types. This makes the code dynamic and scalable.

🎮 Real World Case: The Universal Remote
Imagine a Power Button on a universal remote.

- When you press it for the TV, it turns on the display.
- When you press it for the AC, it starts the cooling fan.
- When you press it for the Music System, it starts the speakers.

The Action (interface) is the same: Press Power. But the Reaction (implementation) is different for every device.

02 The "Overloading" Myth vs Dispatch

Unlike Java or C++, Python does not support traditional method overloading (same method name with different parameters). In Python, the last defined method overwrites everything before it.

Expert Tool
To achieve polymorphism based on data types, Python engineers use @singledispatch from the functools module.
polymorphic_dispatch.py 
from functools import singledispatch

@singledispatch
def process(data):
    print(f"Processing generic data: {data}")

@process.register(int)
def _(data):
    print(f"Processing Integer: {data * 10}")

@process.register(list)
def _(data):
    print(f"Processing List of size: {len(data)}")

process("Hello") # Generic
process(50)      # Int version
process([1, 2, 3]) # List version

03 Duck Typing (Python's Philosophy)

In Python, we don't care about the Type of the object. We care about the Behavior. This is called Duck Typing: "If it walks like a duck and quacks like a duck, it's a duck."

duck_typing_system.py 
class CreditCard:
    def pay(self, amount):
        print(f"Processing ${amount} via Visa/MasterCard.")

class CryptoWallet:
    def pay(self, amount):
        print(f"Processing ${amount} via Blockchain (BTC/ETH).")

# This function accepts ANY object that has a .pay() method
def process_checkout(payment_method, amount):
    payment_method.pay(amount)

# Real-time usage
user_card = CreditCard()
user_crypto = CryptoWallet()

process_checkout(user_card, 1200)   # Polymorphic behavior
process_checkout(user_crypto, 1200) # Polymorphic behavior

04 Runtime Polymorphism (Overriding)

This is also known as Dynamic Polymorphism. The method call is resolved at runtime based on the actual object type, not the variable declaration.

overriding_example.py 
class Shape:
    def area(self): print("Calculating area...")

class Circle(Shape):
    def area(self, radius): print(f"Circle Area: {3.14 * radius**2}")

class Square(Shape):
    def area(self, side): print(f"Square Area: {side * side}")

# The behavior changes based on the OBJECT type
my_shape = Circle()
my_shape.area(5)  # 78.5

05 The "Secret" Polymorphism

Python's built-in operators and functions are the best examples of polymorphism in action.

Operator Polymorphism

The + operator:
- 1 + 2 = 3 (Addition)
- "A" + "B" = "AB" (Concat)
- [1] + [2] = [1, 2] (Merging)

Function Polymorphism

The len() function:
- len("abc") -> 3
- len([1, 2]) -> 2
- len({'a':1}) -> 1
