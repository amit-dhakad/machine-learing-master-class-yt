Supercharging Code with Decorators

A powerful design pattern in Python that allows you to wrap another function in order to extend its behavior, without permanently modifying it.

01 What is a Decorator?

In Python, functions are First-Class Objects. This means they can be passed around and used as arguments, just like strings or integers.

A Decorator takes in a function, adds some functionality (before or after the original function runs), and returns it. We apply a decorator to a function by placing the @decorator_name syntax right above it.

🎁 Real World Case: Gift Wrapping
Imagine you buy a book (your function). A decorator is like a gift wrapping service. You give them the book, they put it in a box, wrap it in nice paper, and give it back to you. It's still the exact same book inside, but its presentation has been extended!

02 Building a Basic Decorator

Let's create a custom decorator called @logger that prints a message before and after a function runs.

basic_decorator.py 
def logger(func):
    def wrapper():
        print("[LOG] Function is about to run...")
        func()  # Execute the original function
        print("[LOG] Function has finished running.")
    return wrapper

@logger
def say_hello():
    print("Hello, World!")

# Calling the decorated function
say_hello()

# Output:
# [LOG] Function is about to run...
# Hello, World!
# [LOG] Function has finished running.

03 Decorating Functions with Arguments

What if our original function takes arguments (like `a` and `b`)? Our wrapper function inside the decorator must be prepared to accept them using *args and **kwargs.

args_decorator.py 
def timer(func):
    import time
    def wrapper(*args, **kwargs):
        start_time = time.time()
        
        # Pass all arguments to the original function
        result = func(*args, **kwargs) 
        
        end_time = time.time()
        print(f"Execution took {end_time - start_time:.4f} seconds")
        
        return result # Don't forget to return the result!
    return wrapper

@timer
def heavy_computation(n):
    return sum(i**2 for i in range(n))

print(heavy_computation(1000000))

04 Real-World Usage: Authentication

Decorators are heavily used in frameworks like Flask or Django to check if a user is logged in before allowing them to access a route.

auth_decorator.py 
current_user = {"is_logged_in": False, "name": "Guest"}

def login_required(func):
    def wrapper(*args, **kwargs):
        if not current_user["is_logged_in"]:
            print("⛔ Access Denied: Please log in first.")
            return
        return func(*args, **kwargs)
    return wrapper

@login_required
def view_dashboard():
    print(f"Welcome to your dashboard, {current_user['name']}!")

# Attempt 1: Not logged in
view_dashboard()

# Attempt 2: Logged in
current_user["is_logged_in"] = True
current_user["name"] = "Alice"
view_dashboard()
