Module 04 · Inheritance

The Legacy of Inheritance

Creating new classes based on existing ones to promote code reusability and establish hierarchical relationships.

01 What is Inheritance?

Inheritance allows a Child Class (Subclass) to derive attributes and methods from a Parent Class (Base class). It enables Code Reusability—you don't have to write the same logic twice.

🧬 Real World Case: Family Genetics
Just as you inherit features like eye color or height from your parents, a Child Class inherits features (methods/variables) from its Parent Class. You can also "evolve" by adding your own unique features or "overriding" inherited ones.

02 Types of Inheritance

Python supports several inheritance patterns depending on how classes are structured.

Single Inheritance One child class inherits from exactly one parent class.
Multiple Inheritance A child class inherits from more than one parent class.
Multi-level Inheritance A child class inherits from a parent, which in turn inherits from another parent (Grandparent).
Hierarchical Inheritance Multiple child classes inherit from a single parent class.

03 Single & Multi-level Inheritance

In Single Inheritance, we use class Child(Parent):. In Multi-level, the chain continues.

levels_demo.py 
class Device: # Grandparent
    def power_on(self):
        print("Device is now ON")

class Phone(Device): # Parent (Single Inheritance)
    def call(self):
        print("Making a call...")

class SmartPhone(Phone): # Child (Multi-level)
    def browse(self):
        print("Browsing the web...")

s = SmartPhone()
s.power_on() # Inherited from Device
s.call()     # Inherited from Phone
s.browse()   # Own method

04 Multiple Inheritance & MRO

A class can inherit from multiple parents. However, if two parents have the same method, Python uses MRO (Method Resolution Order) to decide which one to call.

multiple_inh.py 
class Camera:
    def take_photo(self):
        print("Click! Photo taken.")

class MusicPlayer:
    def play(self):
        print("Playing music...")

class MultimediaPhone(Camera, MusicPlayer):
    pass

mp = MultimediaPhone()
mp.take_photo()
mp.play()
💡 Pro Tip: You can check the MRO of any class using ClassName.mro(). It tells you the exact search path Python follows to find a method.

05 Inheritance + Encapsulation

This is where things get interesting. How do access modifiers behave when inherited?

  • Public: Inherited and accessible normally.
  • Protected (_var): Inherited and accessible in the child class (by convention).
  • Private (__var): NOT directly accessible in the child class due to Name Mangling.
encap_inheritance.py 
class Parent:
    def __init__(self):
        self.public = "I am Public"
        self._protected = "I am Protected"
        self.__private = "I am Private"

class Child(Parent):
    def show_data(self):
        print(self.public)      # OK
        print(self._protected)   # OK (Protected is meant for this!)
        # print(self.__private)  # ERROR! Child can't see this.
        
        # Accessing private via mangled name (Not recommended)
        print(self._Parent__private) # Works!

obj = Child()
obj.show_data()
🛡️ The Shield Rule
Encapsulation ensures that even if you inherit everything, you don't necessarily have the "keys" to the private vaults of your parent. This maintains Data Integrity even across hierarchies.

06 Using super()

The super() function is used to call methods (especially __init__) from the parent class without naming it explicitly.

super_usage.py 
class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

class Manager(Employee):
    def __init__(self, name, salary, department):
        # Use super() to handle parent initialization
        super().__init__(name, salary)
        self.department = department

m = Manager("Alice", 90000, "Engineering")
print(f"{m.name} manages {m.department}")
