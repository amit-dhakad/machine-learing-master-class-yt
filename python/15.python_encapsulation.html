Module 03 · Encapsulation

The Shield of Encapsulation

Wrapping data and code into a single unit to protect the internal state from unauthorized access and accidental modification.

01 What is Encapsulation?

Encapsulation is the practice of bundling data (variables) and methods (functions) that act on that data within a single unit (Class). It also involves Restricting Access to some of an object's components.

💊 Real World Case: The Medical Capsule
Think of a medical capsule. All the medicinal chemicals (data) are enclosed inside a plastic shell (class). You don't interact with the powder directly; you just swallow the capsule. The shell protects the medicine and ensures it is delivered correctly.

02 Access Modifiers in Python

Python doesn't have strict "private" or "public" keywords like Java. Instead, it uses a Naming Convention to signal intended access levels.

var Public Accessible from anywhere (No underscore)
_var Protected Accessible within Class & Subclasses
__var Private Accessible only within the Class
access_demo.py 
class Bank:
    def __init__(self):
        self.bank_name = "Swiss Global"    # Public
        self._branch = "Zurich Main"      # Protected
        self.__vault = 5000000           # Private

acc = Bank()
print(acc.bank_name)  # Works
print(acc._branch)   # Works (but discouraged)
# print(acc.__vault)  # Error! AttributeError

03 Name Mangling

When you use double underscores (__), Python performs Name Mangling. It internally renames the variable to _ClassName__variableName to prevent naming collisions in inheritance.

mangling_trick.py 
acc = Bank()
# Secretly accessing private data via mangled name
print(acc._Bank__vault) # 5000000

04 Pythonic Way: @property

Instead of writing get_balance() and set_balance(), we use the @property decorator. This allows us to use getter/setter logic while keeping the syntax as simple as a variable access.

property_logic.py 
class Employee:
    def __init__(self, salary):
        self.__salary = salary

    @property
    def income(self):
        return f"${self.__salary}"

    @income.setter
    def income(self, new_val):
        if new_val > 0:
            self.__salary = new_val
        else:
            print("Invalid Salary!")

emp = Employee(2500)
emp.income = 3000  # Using setter
print(emp.income) # Using getter (Output: $3000)

05 Why Use Encapsulation?

  • Security: Prevents outside code from corrupting internal data.
  • Maintainability: You can change the internal implementation without affecting the code that uses the object.
  • Flexibility: You can make attributes read-only (by providing a getter but no setter).
  • Validation: You can check if the data is valid before saving it.
