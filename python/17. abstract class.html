Module 05 · Abstract Classes

Designing Blueprints with Abstract Classes

Enforce strict structures and define "what to do" without worrying about "how to do it" using Python's abc module.

01 What is an Abstract Class?

An Abstract Class is a class that contains one or more abstract methods. An abstract method is a method that is declared, but contains no implementation. Abstract classes cannot be instantiated, and require subclasses to provide implementations for the abstract methods.

📐 Real World Case: Architectural Blueprints
Think of an abstract class as a blueprint for a house. The blueprint specifies that a house MUST have a build_roof() and build_walls(), but it doesn't build them itself. It forces any actual house (subclass) created from it to implement those specific construction methods.

02 The abc Module

Unlike some other languages, Python does not have built-in keywords for abstract classes. Instead, we use the built-in abc (Abstract Base Classes) module.

To create an abstract class, you inherit from ABC (Abstract Base Class) and use the @abstractmethod decorator for the methods you want to enforce.

shape_abstract.py 
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass
        
    def describe(self):
        print("I am a geometric shape.")

# The following line would raise a TypeError:
# s = Shape() # ERROR! Cannot instantiate abstract class
⚠️ Common Gotcha: You MUST inherit from ABC (e.g., class Shape(ABC):). If you only use the @abstractmethod decorator but write class Shape():, Python will NOT enforce the abstract methods, and you will still be able to instantiate the class!

03 Implementing the Blueprint

When a child class inherits from an abstract class, it MUST override and implement all the abstract methods. If it doesn't, the child class also becomes abstract and cannot be instantiated.

concrete_shapes.py 
import math
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
        
    # We MUST implement the area method
    def area(self):
        return math.pi * (self.radius ** 2)

class Square(Shape):
    def __init__(self, side):
        self.side = side
        
    # We MUST implement the area method
    def area(self):
        return self.side ** 2

c = Circle(5)
print(f"Circle Area: {c.area():.2f}")

sq = Square(4)
print(f"Square Area: {sq.area()}")
💡 Key Rule: Abstract methods act as a strict contract. The developer writing the Circle class doesn't have a choice; they must provide a way to calculate the area, making large codebases predictable.

04 Real-World Example: Payment Processors

Abstract classes are frequently used in API designs and large systems (like payment gateways) to ensure all plugins/modules adhere to the exact same structure.

payment_gateway.py 
from abc import ABC, abstractmethod

class PaymentProcessor(ABC):
    @abstractmethod
    def authenticate(self):
        pass

    @abstractmethod
    def process_payment(self, amount):
        pass

class StripeProcessor(PaymentProcessor):
    def authenticate(self):
        print("Authenticating with Stripe API...")
        
    def process_payment(self, amount):
        print(f"Processing ${amount} via Stripe.")

class PayPalProcessor(PaymentProcessor):
    def authenticate(self):
        print("Authenticating with PayPal OAuth...")
        
    def process_payment(self, amount):
        print(f"Processing ${amount} via PayPal.")

def checkout(processor: PaymentProcessor, amount):
    processor.authenticate()
    processor.process_payment(amount)

# The system accepts any class that correctly implements the blueprint!
my_processor = StripeProcessor()
checkout(my_processor, 50.00)
